From file to URL in three moves.

02 — How it worksNo dashboard, no waiting room. Your agent writes an artifact, calls one command, hands back a link.

STEP 01

Build an artifact

Your agent writes a .html, .tsx or .vue — a counter, a dashboard, a whole built static folder. Whatever it just generated.

STEP 02

Run one command

conjure deploy counter.tsx auto-wraps the component into a self-contained page and uploads it. No build step on your end.

STEP 03

Share the link

Out comes a clean public URL. No viewer login, no cold-start tax — the backend scales to zero between hits.

Small surface. Serious infrastructure.

03 — What you getA tiny CLI on top of a single Cloudflare Worker and R2 bucket — running entirely in your own account.

F/01

One Worker + R2

No database. No always-on server. A single Cloudflare Worker fronts an R2 bucket, so idle cost rounds to ~$0 and there's almost nothing to operate.

F/02

Agent-native

Drop a snippet in AGENTS.md and any coding agent can publish on its own — no human in the deploy loop.

F/03

Auto-wrap

Hand it one .tsx/.vue/.html and it becomes a runnable page — React 18 + Babel + Tailwind, or Vue 3, all self-contained.

F/04

Yours to host

Runs in your Cloudflare account, with your token. Free software under MIT — fork it, audit it, run it forever. No vendor between you and your links.
