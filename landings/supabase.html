// how it works

Three steps to a shareable link

From an agent's output file to a public URL — no build pipeline, no deploy config, no waiting on a server to spin up.

1

Build an artifact

Your agent writes a .html, .tsx, or .vue — or a built static folder. Whatever it generates, Conjure can ship it.

2

Run one command

conjure deploy auto-wraps single components into self-contained pages and uploads the result in a single call.

3

Share the link

You get a live URL instantly. No viewer login, no cold-start lag — and it scales right back down to zero when idle.

Built for agents. Owned by you.

One Cloudflare Worker, one R2 bucket, and a tiny CLI. That's the whole stack.

One Worker + R2

No database. No always-on server. A single Cloudflare Worker fronts an R2 bucket, so idle cost rounds to about $0.

~$0 idle

Agent-native

Drop a snippet into AGENTS.md and any coding agent can publish its own work — no human in the loop required.

conjure deploy <file>
