Three steps

How it works

  1. 01

    Build an artifact

    Your agent writes a single file — an .html page, a React .tsx, or a Vue .vue component. No project, no config, no build pipeline required.

  2. 02

    Run one command

    getonup deploy auto-wraps the component into a self-contained page and uploads it straight to your R2 bucket. The whole round trip takes seconds.

  3. 03

    Share the link

    You get back a clean, permanent URL. No viewer login, nothing to keep running — the backend scales to zero when nobody's looking.

Why it stays simple

What you get

One Worker + R2

No database, no always-on server. A single Cloudflare Worker fronts an R2 bucket — idle cost rounds to ~$0.

Agent-native

Drop a snippet in AGENTS.md and any coding agent can publish its own work, no human in the loop.

Auto-wrap

Hand it one .tsx or .vue and it becomes a runnable page — React 18 + Babel + Tailwind, or Vue 3, fully self-contained.

Yours to host

Your Cloudflare account, your token, your data. Free and open source under the MIT license.

Questions

? FAQ

How does auto-wrap work?+
Pass a bare component and getonup detects the framework from the file. A .tsx is wrapped in a page that pulls React 18, Babel for in-browser transform, and Tailwind; a .vue becomes a Vue 3 page. Plain .html is uploaded as-is. The result is one self-contained file that runs anywhere — no bundler on your machine.
Where do my Cloudflare token and secrets live?+
Entirely on your side. getonup deploys to your Worker and your R2 bucket using a token you create and store locally. Nothing routes through a third party — there's no getonup-operated cloud to trust. The CLI only talks to the endpoint you configured.
What does self-hosting on Cloudflare involve?+
Deploy one Worker, create one R2 bucket, set a token. That's the whole stack — no database, no container, nothing to babysit. Because the Worker scales to zero, an idle instance costs essentially nothing, and traffic rides Cloudflare's free-tier limits for a long while.
Can coding agents publish on their own?+
Yes — that's the point. Add the getonup deploy snippet to your AGENTS.md and any agent that reads it can ship a preview link the moment it finishes writing an artifact. The link comes back on stdout, ready to paste into a PR, a chat, or a review.
