How it works

Three steps, zero ceremony.

From a file your agent just wrote to a public link anyone can open.

1

Build an artifact

Your agent writes a .html, .tsx or .vue file. No project setup, no scaffolding — just the artifact itself.

2

Run one command

Call getonup deploy <file>. It auto-wraps your component into a self-contained page and uploads it.

3

Share the link

Get a live URL back instantly. No viewer login, no cold-start server — it scales to zero when idle.

Why getonup

Built for agents. Owned by you.

A tiny CLI any coding agent can call, backed by infrastructure that costs roughly nothing at rest.

One Worker + R2

No database, no always-on server. A single Cloudflare Worker fronting R2 object storage that scales to zero — roughly $0 idle.

Agent-native

Drop a snippet in your AGENTS.md so any coding agent can publish on its own. One command, no human in the loop.

Auto-wrap

Hand it one .tsx/.vue/.html — it becomes a runnable page. React 18 + Babel + Tailwind, or Vue 3, all self-contained.

Yours to host

Runs in your Cloudflare account, with your token, under your domain. Free and MIT-licensed — no platform lock-in, ever.
