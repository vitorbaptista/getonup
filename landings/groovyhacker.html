How it works

Three moves and you're on the air. No build step, no babysitting.

  1. 1

    Your agent builds the artifact

    A coding agent writes a single file — an .html page, a React .tsx, or a Vue .vue component. No project scaffolding, no config.

  2. 2

    Run getonup deploy <file>

    getonup auto-wraps a bare component into a runnable page and pushes it straight to your R2 bucket. The whole round trip takes seconds.

  3. 3

    Share the live link

    You get back a clean, shareable URL — paste it into a PR, a chat, or a review. No viewer login, nothing left running.

What you get

A whole publishing stack that fits in one Worker — and it's yours.

WORKER + R2

Scales to zero

One Cloudflare Worker fronts an R2 bucket. No database, no always-on server — idle cost rounds to ~$0.

AGENT-NATIVE

Built for coding agents

Drop a line in AGENTS.md and any agent can publish its own work and hand back a link — no human in the loop.

AUTO-WRAP

React / Vue / TS, no build

Hand it one bare component and it becomes a self-contained page — React + Tailwind or Vue 3, transformed in the browser.
