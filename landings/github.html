How it works

From artifact to shareable link in three steps.

No dashboards, no deploy pipelines to configure. Hand getonup a file and get a URL back.

1

Build an artifact

Your agent writes a .html, .tsx, or .vue file — a counter, a dashboard, a landing page. Whatever it just generated.

2

Run one command

getonup deploy auto-wraps a single component into a self-contained page, then uploads it to your Worker and R2 bucket.

3

Share the link

You get a live URL instantly. No viewer login, no cold-start wait — it scales to zero when nobody's looking.

Features

Built to be small, owned, and agent-native.

One Worker and one bucket do the whole job. No moving parts to babysit.

One Worker + R2

No database, no always-on server, ~$0 idle. A single Cloudflare Worker serves pages straight from an R2 bucket and scales all the way down to zero.

Agent-native

Drop a snippet in AGENTS.md so any coding agent can publish on its own. getonup is one CLI call any agent can reach for the moment it finishes a build.

Auto-wrap

Hand it one .tsx / .vue / .html and it becomes a runnable page — React 18 with Babel and Tailwind, or Vue 3 — self-contained, no build step.

Yours to host

Runs in your Cloudflare account, with your token. The whole thing is free software under the MIT license — fork it, audit it, keep it.

Self-host getonup in minutes.

Deploy the Worker to your own Cloudflare account, point the CLI at it, and start shipping links. Open source, end to end.
