How it works

From file to live URL in three moves

No dashboard, no build pipeline, no waiting. One command does the wrapping, the upload, and the link.

1

Build an artifact

Your agent writes a .html, .tsx, or .vue — or hands you a built static folder. Whatever it just generated, getonup can ship.

2

Run one command

getonup deploy auto-wraps single components into a self-contained page, then uploads to your Worker + R2 backend.

3

Share the link

You get a clean public URL instantly. No viewer login, nothing to keep running — it scales to zero when idle.

Built to be tiny

One Worker. One bucket. Yours.

A single Cloudflare Worker fronts R2 object storage. That's the whole backend — agent-native, auto-wrapping, and pennies at rest.

One Worker + R2

No database, no always-on server, no cold-start tax to pay all month. A single Worker plus object storage means roughly $0 while nobody's looking.

Agent-native

Drop a short snippet in your AGENTS.md and any coding agent can publish its own work. getonup is a tool agents call, not a UI humans babysit.

Auto-wrap

Hand it one .tsx, .vue or .html and it becomes a runnable page — React 18 + Babel + Tailwind, or Vue 3, all self-contained.

Yours to host

Runs on your Cloudflare account with your token — no third party in the loop. MIT licensed, fully self-hostable, and free software end to end.

getonup it onto your own infra

Clone the repo, point it at your Cloudflare account, and give every agent in your stack a live URL — in seconds, at scale-to-zero cost.
