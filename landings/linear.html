How it works

Artifact to URL in three steps

No dashboard, no build pipeline, no waiting. Hand it a file and share the link.

1

Build an artifact

Your agent writes a .html, .tsx, or .vue — or a built static folder. Whatever it just generated is ready to go.

2

Run one command

cjr deploy auto-wraps your component into a self-contained page and uploads it. No config to fill in.

3

Share the link

You get a live URL instantly. No viewer login, no cold-start tax — it scales to zero when idle.

Built different

A backend that does less, on purpose

One Cloudflare Worker, one R2 bucket, and a CLI any agent can call. That's the whole stack.

One Worker + R2

No database, no always-on server, no infrastructure to babysit. It costs roughly $0 while idle and only spins up when someone opens a link.

Agent-native

Drop a snippet into AGENTS.md and any coding agent can publish on its own — deploy becomes part of the workflow, not a manual step.

Auto-wrap

Hand it one .tsx or .vue and it becomes a runnable page — React 18 + Babel + Tailwind, or Vue 3, wired up and served as one self-contained file.

Yours to host

Your Cloudflare account, your token, your domain. The whole thing is MIT-licensed and self-hostable — nothing routes through a service you don't control.

Spin up your own in minutes

Clone it, point it at your Cloudflare account, and start conjuring live links. Free software, MIT.
