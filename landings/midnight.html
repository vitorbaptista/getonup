// three steps

From file to live link, no detours.

No dashboard, no build pipeline, no waiting on a server to spin up. Just point Conjure at an artifact.

01

Build an artifact

Your agent writes a .html, .tsx, or .vue — or hands you a whole built static folder. Anything web-shaped works.

02

Run one command

Call cjr deploy <file>. It auto-wraps single components into a self-contained page and uploads to your R2.

03

Share the link

Get a clean URL back instantly. No viewer login, no cold-start tax — the Worker scales straight back to zero when idle.

// why conjure

Minimal infrastructure. Maximum range.

Built to be invisible to your agents and nearly free to run for you.

One Worker + R2

No database, no always-on server, no queue. A single Cloudflare Worker and an R2 bucket do the whole job — and sit at ~$0 idle because everything scales to zero.

Agent-native

Drop a snippet in your AGENTS.md and any coding agent can publish on its own. One terminal call — no SDK, no auth dance, no human in the loop.

Auto-wrap

Hand it one .tsx and it becomes a runnable page — React 18 + Babel + Tailwind. Vue 3 components too. Plain .html ships as-is. Zero boilerplate from you.

Yours to host

Your Cloudflare account, your token, your domain. No vendor lock-in, no usage caps you didn't set. Free software under the MIT license — fork it, own it.
