How it works

Three steps from file to live link.

No dashboard, no build pipeline, no waiting on a server to spin up. Hand getonup one file and it does the rest.

01

Build an artifact

Your agent writes a .html, .tsx, or .vue — a counter, a chart, a landing page, whatever it just dreamt up.

02

Run one command

getonup deploy auto-wraps single components into a self-contained page and uploads it to your R2 bucket in seconds.

03

Share the link

You get a clean, public URL. No viewer login, no cold-start tax — and it scales straight back to zero when nobody's looking.

What you get

Tiny footprint. Big reach.

getonup is deliberately small — one Worker, one bucket, no database — so it stays cheap, fast, and entirely yours.

One Worker + R2

No database, no always-on server. A single Cloudflare Worker serves every artifact straight from R2. Idle cost rounds to ~$0.

Agent-native

Drop one snippet in AGENTS.md and any coding agent can publish on its own. getonup was built to be called by a machine, not clicked by a human.

Auto-wrap

Hand it one .tsx, .vue, or .html and it becomes a runnable page — React 18 + Babel + Tailwind, or Vue 3, bundled self-contained.

Yours to host

Runs on your Cloudflare account with your token. No middleman, no lock-in, no surprise bill — and it's MIT licensed, all the way down.
