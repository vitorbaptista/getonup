How it works

Three steps. One command.

From a file your agent just wrote to a link anyone can open — no dashboard, no viewer login, nothing to keep warm.

1

Build an artifact

Your agent writes a .html, .tsx, or .vue — or points at a built static folder. Whatever it generated, getonup can ship it.

counter.tsx
2

Run one command

getonup deploy auto-wraps single components into a self-contained page, then uploads straight to your storage.

$ getonup deploy
3

Share the link

You get a clean, live URL in seconds. No viewer login, no cold-start wait — and it scales all the way down to zero when idle.

/s/iwcmg3dt
Why getonup

Built for agents and the developers who run them.

A tiny CLI on top of a single Cloudflare Worker and R2 bucket. That's the whole stack.

One Worker + R2

No database. No always-on server. A single Cloudflare Worker handles routing and an R2 bucket holds your artifacts — so it sits at roughly $0 when nobody's looking.

~$0 idle · no infra to maintain

Agent-native

Drop a short snippet into your AGENTS.md and any coding agent can publish on its own. getonup is a command, not a console — exactly what an agent knows how to call.

one command, any coding agent

Auto-wrap

Hand it one .tsx, .vue, or .html and it becomes a runnable page — React 18 + Babel + Tailwind, or Vue 3, bundled into a self-contained document.

component in, live page out

Yours to host

Runs on your Cloudflare account, with your token, under your domain. It's free software (MIT) — fork it, change it, keep every byte of your data where you want it.

your account · your token · MIT
