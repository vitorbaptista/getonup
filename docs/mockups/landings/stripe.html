How it works

From file to live URL in three moves.

Your agent writes the artifact. getonup handles the rest.

1

Build an artifact

Your coding agent writes a .html, .tsx, or .vue file — a counter, a dashboard, a whole built static folder.

2

Run one command

Call getonup deploy. It auto-wraps single components into a runnable page and uploads in seconds.

3

Share the link

Get a live URL back. No viewer login, no cold-start wait — and it scales to zero when idle.

Why getonup

Built lean. Built for agents.

One Worker, one bucket, and a CLI any agent can call.

One Worker + R2

No database, no always-on server. A single Cloudflare Worker fronts an object store and scales to zero — roughly $0 idle.

Agent-native

Drop a snippet into AGENTS.md and any coding agent can publish a live preview the moment it finishes writing code.

Auto-wrap

Hand it one .tsx or .vue and it becomes a self-contained page — React 18 + Babel + Tailwind, or Vue 3. Plain HTML ships as-is.

Yours to host

Your Cloudflare account, your token, your domain. Free and MIT licensed — fork it, audit it, run it forever.
