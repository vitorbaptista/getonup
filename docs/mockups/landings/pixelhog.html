From file to live URL.

Skip the viewer logins and dashboard wrangling. Your agent writes a file and getonup does the rest.

1

Build an artifact

Your coding agent writes a .html, .tsx, or .vue. A single component is plenty, with no boilerplate or bundler config to set up.

2

Run one command

getonup deploy <file> auto-wraps the component into a self-contained page and uploads it straight to your bucket.

3

Share the link

You get a clean URL on your own domain with no viewer login, and it scales to zero. Share it, screenshot it, or embed it anywhere.

One Worker + R2, scale-to-zero

There's no database and no always-on server to babysit. A single Cloudflare Worker serves while one R2 bucket stores, and it scales to zero, so idle cost is basically nothing. The whole control plane is two files you can read in an afternoon.

~$0 idle
